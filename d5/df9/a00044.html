Library debuggers

Functions pertaining to Thallium debugging. More...

Detailed Description

Library compounds for configuring Thallium debug output with debuggers.

Note that these are independent of the graphics API - for instance, you would still need to handle a Vulkan debug messenger pipeline yourself.

Type Documentation

◆ th_Debugger_t

typedef struct th_Debugger_t th_Debugger_t

This structure can be created and passed to renderer creation to enable runtime debug messages - for instance, it is needed to recieve errors and warnings.

Note that this is independent of API-specific debugging methods (such as OpenGL debug output).

Warning
Debuggers will not be used if the Thallium debug layer was not enabled on compilation.

◆ th_TriggerSeverityMessages()

const void th_TriggerSeverityMessages ( const th_Debugger_t *  debugger)

This function posts a debug message to debugger for each Thallium severity and message type. This can be used to test the debugger's severity configuration.

Warning
Because this function may output a fatal error (depending on the debug types filter), it halts program execution! It is only intended to be temporarily used in development!
Parameters
debuggerDebugger object to which the messages will be posted.
